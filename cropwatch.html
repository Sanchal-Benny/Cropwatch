🧠 Backend Logic

The invisible engine that powers CropWatch — explained visually

🗄️ Database — PostgreSQL + PostGIS
Stores all drone scan data, NDVI results, crop stress alerts, and field zone boundaries with geospatial precision. PostGIS enables spatial queries like "all critical zones within 20km of an irrigation canal."
1
Drone confirms upload → FastAPI saves S3 URL + metadata to PostgreSQL
2
PostGIS stores crop zone geometry as Polygon shapes with GPS boundaries
3
Spatial query: SELECT zones WHERE ndvi_score < 0.4 AND ST_Within(zone, field)
4
Results served to frontend via FastAPI REST endpoint
⚙️ Processing Engine — Celery + NumPy
NDVI computation is CPU-heavy. Celery queues jobs via Redis so the API stays responsive. Workers run in parallel — one file per worker. Progress streams back via WebSocket.
1
Upload confirmed → Celery job enqueued via Redis broker
2
Worker loads GeoTIFF using Rasterio — reads NIR and Red spectral bands
3
NumPy computes: NDVI = (NIR − Red) / (NIR + Red) per pixel across field
4
Heatmap PNG saved to S3 → WebSocket pushes progress to dashboard
🚨 Crop Stress Alert Engine — ML + Rules
Compares current NDVI against a rolling 60-day crop baseline. A scikit-learn classifier identifies drought vs. pest vs. waterlogging patterns. Triggers SMS + email to field officers.
1
Fetch baseline NDVI for the same GPS zone from last 60 days
2
Calculate delta — if NDVI drop > 0.2 within 7 days → flag CRITICAL
3
scikit-learn classifier checks spatial texture pattern (drought / pest / flood)
4
Alert written to DB → push via SendGrid + Twilio + WebSocket
🔌 REST API — FastAPI
Auto-documented REST API with Swagger UI at /docs. Frontend calls these endpoints. JWT authentication on every route. CORS configured for the React dev server.
POST/uploads/presignGenerate S3 URL
POST/uploads/confirmTrigger NDVI job
GET/dronesFleet telemetry
GET/analysis/{id}NDVI results
GET/regions/geojsonZone boundaries
GET/alertsActive stress alerts
GET/analytics/trendsTime-series NDVI
DEL/uploads/{id}Delete + S3 cleanup
